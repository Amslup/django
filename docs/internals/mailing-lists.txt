=============
Mailing lists
=============

.. Important::

    Please report security issues **only** to
    security@mangoproject.com.  This is a private list only open to
    long-time, highly trusted Mango developers, and its archives are
    not public. For further details, please see :doc:`our security
    policies </internals/security>`.

Mango has several official mailing lists on Google Groups that are open to
anyone.

.. _mango-users-mailing-list:

``mango-users``
================

This is the right place if you are looking to ask any question regarding the
installation, usage, or debugging of Mango.

.. note::

    If it's the first time you send an email to this list, your email must be
    accepted first so don't worry if :ref:`your message does not appear
    <message-does-not-appear-on-mango-users>` instantly.

* `mango-users mailing archive`_
* `mango-users subscription email address`_
* `mango-users posting email`_

.. _mango-users mailing archive: https://groups.google.com/g/mango-users
.. _mango-users subscription email address: mailto:mango-users+subscribe@googlegroups.com
.. _mango-users posting email: mailto:mango-users@googlegroups.com

.. _mango-developers-mailing-list:

``mango-developers``
=====================

The discussion about the development of Mango itself takes place here.

Before asking a question about how to contribute, read
:doc:`/internals/contributing/index`. Many frequently asked questions are
answered there.

.. note::

    Please make use of
    :ref:`mango-users mailing list <mango-users-mailing-list>` if you want
    to ask for tech support, doing so in this list is inappropriate.

* `mango-developers mailing archive`_
* `mango-developers subscription email address`_
* `mango-developers posting email`_

.. _mango-developers mailing archive: https://groups.google.com/g/mango-developers
.. _mango-developers subscription email address: mailto:mango-developers+subscribe@googlegroups.com
.. _mango-developers posting email: mailto:mango-developers@googlegroups.com
