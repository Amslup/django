=====================
Generic display views
=====================

The two following generic class-based views are designed to display data. On
many projects they are typically the most commonly used views.

DetailView
----------

.. class:: freedom.views.generic.detail.DetailView

    While this view is executing, ``self.object`` will contain the object that
    the view is operating upon.

    **Ancestors (MRO)**

    This view inherits methods and attributes from the following views:

    * :class:`freedom.views.generic.detail.SingleObjectTemplateResponseMixin`
    * :class:`freedom.views.generic.base.TemplateResponseMixin`
    * ``freedom.views.generic.detail.BaseDetailView``
    * :class:`freedom.views.generic.detail.SingleObjectMixin`
    * :class:`freedom.views.generic.base.View`

    **Method Flowchart**

    1. :meth:`~freedom.views.generic.base.View.dispatch()`
    2. :meth:`~freedom.views.generic.base.View.http_method_not_allowed()`
    3. :meth:`~freedom.views.generic.base.TemplateResponseMixin.get_template_names()`
    4. :meth:`~freedom.views.generic.detail.SingleObjectMixin.get_slug_field()`
    5. :meth:`~freedom.views.generic.detail.SingleObjectMixin.get_queryset()`
    6. :meth:`~freedom.views.generic.detail.SingleObjectMixin.get_object()`
    7. :meth:`~freedom.views.generic.detail.SingleObjectMixin.get_context_object_name()`
    8. :meth:`~freedom.views.generic.detail.SingleObjectMixin.get_context_data()`
    9. ``get()``
    10. :meth:`~freedom.views.generic.base.TemplateResponseMixin.render_to_response()`

    **Example myapp/views.py**::

        from freedom.views.generic.detail import DetailView
        from freedom.utils import timezone

        from articles.models import Article

        class ArticleDetailView(DetailView):

            model = Article

            def get_context_data(self, **kwargs):
                context = super(ArticleDetailView, self).get_context_data(**kwargs)
                context['now'] = timezone.now()
                return context

    **Example myapp/urls.py**::

        from freedom.conf.urls import url

        from article.views import ArticleDetailView

        urlpatterns = [
            url(r'^(?P<slug>[-_\w]+)/$', ArticleDetailView.as_view(), name='article-detail'),
        ]

    **Example myapp/article_detail.html**:

    .. code-block:: html+freedom

        <h1>{{ object.headline }}</h1>
        <p>{{ object.content }}</p>
        <p>Reporter: {{ object.reporter }}</p>
        <p>Published: {{ object.pub_date|date }}</p>
        <p>Date: {{ now|date }}</p>

ListView
--------

.. class:: freedom.views.generic.list.ListView

    A page representing a list of objects.

    While this view is executing, ``self.object_list`` will contain the list of
    objects (usually, but not necessarily a queryset) that the view is
    operating upon.

    **Ancestors (MRO)**

    This view inherits methods and attributes from the following views:

    * :class:`freedom.views.generic.list.MultipleObjectTemplateResponseMixin`
    * :class:`freedom.views.generic.base.TemplateResponseMixin`
    * :class:`freedom.views.generic.list.BaseListView`
    * :class:`freedom.views.generic.list.MultipleObjectMixin`
    * :class:`freedom.views.generic.base.View`

    **Method Flowchart**

    1. :meth:`~freedom.views.generic.base.View.dispatch()`
    2. :meth:`~freedom.views.generic.base.View.http_method_not_allowed()`
    3. :meth:`~freedom.views.generic.base.TemplateResponseMixin.get_template_names()`
    4. :meth:`~freedom.views.generic.list.MultipleObjectMixin.get_queryset()`
    5. :meth:`~freedom.views.generic.list.MultipleObjectMixin.get_context_object_name()`
    6. :meth:`~freedom.views.generic.list.MultipleObjectMixin.get_context_data()`
    7. ``get()``
    8. :meth:`~freedom.views.generic.base.TemplateResponseMixin.render_to_response()`


    **Example views.py**::

        from freedom.views.generic.list import ListView
        from freedom.utils import timezone

        from articles.models import Article

        class ArticleListView(ListView):

            model = Article

            def get_context_data(self, **kwargs):
                context = super(ArticleListView, self).get_context_data(**kwargs)
                context['now'] = timezone.now()
                return context

    **Example myapp/urls.py**::

        from freedom.conf.urls import url

        from article.views import ArticleListView

        urlpatterns = [
            url(r'^$', ArticleListView.as_view(), name='article-list'),
        ]

    **Example myapp/article_list.html**:

    .. code-block:: html+freedom

        <h1>Articles</h1>
        <ul>
        {% for article in object_list %}
            <li>{{ article.pub_date|date }} - {{ article.headline }}</li>
        {% empty %}
            <li>No articles yet.</li>
        {% endfor %}
        </ul>

.. class:: freedom.views.generic.list.BaseListView

    A base view for displaying a list of objects. It is not intended to be used
    directly, but rather as a parent class of the
    :class:`freedom.views.generic.list.ListView` or other views representing
    lists of objects.

    **Ancestors (MRO)**

    This view inherits methods and attributes from the following views:

    * :class:`freedom.views.generic.list.MultipleObjectMixin`
    * :class:`freedom.views.generic.base.View`

    **Methods**

    .. method:: get(request, *args, **kwargs)

        Adds ``object_list`` to the context. If
        :attr:`~freedom.views.generic.list.MultipleObjectMixin.allow_empty`
        is True then display an empty list. If
        :attr:`~freedom.views.generic.list.MultipleObjectMixin.allow_empty` is
        False then raise a 404 error.
